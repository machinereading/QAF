What is FRDF?

For the KB-QA, two core modules are required. First, the query generation module, which understans natural language query (NLQ) and converts it to machine readable query (e.g. SPARQL query). Secend, the knolwedge extraction module, which understand natural language text in the web and converts it to machine readable structured data (e.g. RDF triple). However, in the KB-QA respective, ontological representation of knowledge (called like 'typed relation', 'schema', 'T-Box', and so son) would use restricted vocabulary. For example, DBpedia ontology T-Box is based on the infobox in the Wikipedia. It would be modified to represent simple facts which be suit to encyclopedia's purpose, such as NAME, JOB, LOCATION, AGE, QUANTITY, ...(Hahm et al., 2014). T-Box-based natural language understanding approach (called top-down) would be limited because of its domain scope. For example, to understand the NLQ "Who is the first person reached the South Pole?" in the scope of DBpedia T-Box, the predicate 'reached' would be converted as dbo:knownFor, and the quantifier word 'first' could not be dealt with.

FRDF uses the semantic frames vocabulary in the field of linguistics (FrameNet). Semantic frames represent not only factual information, but also linguistic level semantics, such as CAUSE & EFFECT, EMOTION, OPINION, MOTION, PROBLEM & SOLUTION, and so on. These linguistic level semantics would be used for bottom-up knowledge grounding to understand natural language query and text in KB-QA perspective.

FRDF consists of two major modules: 1) QAF - converts natural language question (NLQ) to SPARQL query, and 2) FRDF-KB - extracts knowledge from text and converts it to RDF KB. At this time (August, 2016), only QAF module is published on our github repository.

For more information please visit: https://github.com/machinereading/FRDF.
