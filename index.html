What is FRDF?

For the KB-QA, two core modules are required. First, it is the query generation module, which understans natural language query (NLQ) and converts it to machine readable query (e.g. SPARQL query). Secend one is the knolwedge extraction module, which understand natural language text in the web and converts it to machine readable structured data (e.g. RDF triple). However, in the KB-QA respective, ontological representation of knowledge (called like 'typed relation', 'schema', 'T-Box', and so son) would use restricted vocabulary. For example, DBpedia ontology T-Box is based on the infobox in the Wikipedia. It would be modified to represent simple facts which be suit to encyclopedia's purpose, such as NAME, JOB, LOCATION, AGE, QUANTITY, ...(Hahm et al., 2014). T-Box-based natural language understanding approach (called top-down) would be limited because of its domain scope. For example, to understand the NLQ "Who is the first person reached the South Pole?" in the scope of DBpedia T-Box, the predicate 'reached' would be converted as dbo:knownFor, and the quantifier word 'first' could not be dealt with.

FRDF uses the semantic frames vocabulary in the field of linguistics (FrameNet). Semantic frames represent not only factual information, but also linguistic level semantics, such as CAUSE & EFFECT, EMOTION, OPINION, MOTION, PROBLEM & SOLUTION, and so on. These linguistic level semantics would be used for bottom-up knowledge grounding to understand natural language query and text in KB-QA perspective.

FRDF consists of two major modules: 1) QAF - converts natural language question (NLQ) to SPARQL query, and 2) FRDF-KB - extracts knowledge from text and converts it to RDF KB. At this time (August, 2016), only QAF module is published on our github repository.

For more information please visit: https://github.com/machinereading/FRDF.

Module I/O Specification

LIBRARY 1: requestService

how to use: import requestService

class: Parser
method: espresso

Description: This is the ESPRESSO parser, including morphological analyzer, dependency parser, SRL, NER, WSD, and co-reference resolution. For more information visit here

Usage Example:
parser = requestService.Parser()
nlpResult = parser.espresso(str)

  • input: string: input would be a one sentence by sentence
  • output: json: output is json format

method: etri

Description: This is the ETRI parser, including morphological analyzer, dependency parser, SRL, NER, WSD, and co-reference resolution. (Soojong Lim et al., 2014)

Usage Example:
parser = requestService.Parser()
nlpResult = parser.etri(str)

  • input: string: input would be a one sentence by sentence
  • output: json: output is json format

class: Stemmer

method: stem

Description: Stemmer

Usage Example:
stemmer = requestService.Stemmer()
stem = stemmer(str)

  • input: string: morphological analyzed word. e.g. 해전/NNG+은/JX
  • output: string: stem. e.g. 해전

class: WSD

method: disambiguator

Description: WSD tag extractor from JSON format NLP result

Usage Example:
wsd = requestService.WSD()
morp = nlpResult[0]["morp"]
wsd = nlpResult[0]["WSD"]
wsdDict = wsd.disambiguator(str,morp,wsd)

  • input:
    string: word. e.g. 해전
    morp: morphological analyzer result from Korean NLP tool. It is a part of JSON.
    wsd: WSD result from Korean NLP tool. It is a part of JSON.
  • output: dictionary: e.g. {"text":"해전", "pos":"NNG", "ws_code":"03"}
